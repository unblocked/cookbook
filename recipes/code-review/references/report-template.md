# Review Report Template

Adapt this structure to fit the PR. Not every section is needed for every review.

## Summary

One-paragraph assessment. Include an overall recommendation: Approve / Request Changes / Needs Discussion.

## Findings

Group by category (Pattern Mismatch > Reinvented Wheel > Convention Drift > Missing Context > Risk). For each finding include:

- What the code does
- What the team context shows (with citation — PR number, decision, convention)
- Suggested action

## What the PR Does Well

Call out where the PR aligns with team patterns. This confirms the author's good instincts and helps reviewers distinguish intentional choices from oversights.

## Context for the Author

Relevant context the author might not have had when writing the PR — recent changes in the area, related decisions, constraints from dependent systems.
