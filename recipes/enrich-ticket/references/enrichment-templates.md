# Enrichment Templates

Use the template matching the classified mode. Adapt sections to the ticket — not every section applies to every ticket.

---

## Engineering Enrichment

### Affected Systems
Components, services, and code paths with file references.

### Relevant Code
Key files, entry points, and utilities the implementer should know about.

### Related Work
PRs, tickets, and decisions touching this area. Include summaries — not just links.

### Historical Context
Prior attempts, rejected approaches, known constraints. What was tried and what happened.

### Suggested Direction
Based on team patterns — not a full plan, just enough orientation so the implementer doesn't start from zero.

---

## Support Enrichment

### Product Behavior
How the feature works and its known limitations.

### User Impact
Scope of the issue and workarounds if any exist.

### Related Incidents
Similar past reports and their resolutions.

### Resolution History
How the team has handled this type of issue before.

### Ownership
Which team owns this area and domain experts if surfaced by research.
