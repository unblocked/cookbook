# Plan Template

Adapt this structure to the task. The plan should be self-contained — readable and actionable without access to Unblocked or additional context.

## Task Description

What to implement and why.

## Embedded Conventions

Team patterns with code examples pasted inline. Show what the team's code looks like — don't describe it abstractly.

## Embedded Prior Art

Relevant PRs and decisions with key details. Include PR numbers, what they changed, and why it matters for this task.

## File Map

Exact files to create or modify. Verify paths against the codebase.

## Step-by-Step Approach

Specific enough for a context-blind agent or engineer to execute without further research.

## Constraints

What NOT to do — based on rejected approaches, known gotchas, and team decisions discovered during hydration.

## Verification Steps

How to confirm correctness after implementation. Concrete and testable.

---

## Completeness Checklist

Before presenting to the user, verify:

- [ ] Every convention shown with example, not just described
- [ ] All relevant PRs summarized with key details
- [ ] File paths exact and verified against the codebase
- [ ] Constraints explicit
- [ ] Verification steps concrete and testable
